# Product Requirements Docs (PRDs)

## What Is a PRD?

A PRD frames a feature from the **user's perspective**: the problem for users, the user stories, and the acceptance criteria that define "done." It answers *what the product should do and for whom*, leaving *how it is built* to the spec (see `../specs/`).

Write a PRD when a feature needs explicit user-facing requirements before design begins — user stories, acceptance criteria, and scope boundaries.

PRDs are not engineering specs and not implementation plans. They answer: "What should this do for users, and how do we know it's done?"

Create one with `reins prd create "title"` — never hand-write files in this directory.
